## Architecture overview

### Navigation
Bottom nav: HABITS | WORK | CHAR | DUNGEON | LOG
CHAR tab has sub-tabs: STATS | WORK | CLASS
Within CHAR > WORK there is a toggle: **вљ’ WORKBENCH** | **рџЏЄ MY SHOP** (`workShopView` global)

### State shape (key fields)
```js
state = {
  habitos: [],          // daily habits
  weekly: [],           // weekly habits
  trabajo: [],          // work tasks (shop economy)
  negativos: [],        // avoid habits
  stats: {},            // STR/DEX/CON/INT/VOL/CHA
  totalXp: 0,
  gold: 0,              // spendable gold (shop economy)

  dungeon: { ...dungeonFields, activeWorkTaskId, forgeTaskPomoCount } | null,
  // ... achievements, classData, inventory, equipment, etc.

```

### Work task shape
```js
{
  id, text, xp,
  gains: {},            // always empty for trabajo вЂ” no stat gains
  drops: { gold, matId, matEmoji, matLabel, matQty },

  difficulty: 'easy'|'normal'|'hard'|'legendary',
  done: { [dateKey]: { gains:{}, xp, drops } }
}
```

## Shop Heroes redesign вЂ” COMPLETED steps

### Step 1 вЂ” Decoupled economy
Work tasks no longer give stats. They give **gold + materials** via `applyWorkDrops(drops, mul)`.
- `WORK_MATERIALS`: timber(physical), scroll(mental), crystal(creative), gear(operational)
- `TASK_DIFF_CFG`: easyв†’5g/1mat, normalв†’12g/2mat, hardв†’25g/3mat, legendaryв†’50g/5mat
- `getWorkTaskDrops(categoryId, difficultyId)` в†’ `{ gold, matId, matQty, matEmoji, matLabel }`
- Reversal on uncheck and Reset Day both call `applyWorkDrops(c.drops, -1)`

### Step 2 вЂ” Material drops on completion
- `toggleQuest` branches on `listKey==='trabajo'`: calls `applyWorkDrops` instead of `applyGainsWithClass`
- Completion stored as `task.done[periodKey] = { gains:{}, xp, drops }`
- Notification: `+20 EXP В· рџЄ™12g В· рџ“њГ—2`

### Step 3 вЂ” Commission board
Daily 3-commission board generated by `generateCommissions(dateKey)`, stored in `state.commissions`.
- `COMMISSION_POOL`: 14 templates (categoryГ—2, categoryГ—3, difficulty-gated, bulk)
- `updateCommissionProgress(periodKey)`: called in `toggleQuest` for trabajo tasks; awards bonus gold+mat when threshold reached; completed commissions lock permanently
- Renders in `renderCommissionBoard()` between the divider and task list
- Commissions regenerate at midnight (checked on render)

### Step 4 вЂ” MY SHOP view
`workShopView` global toggles between `'workbench'` and `'shop'`.
- `getShopLevel()`: 8 levels keyed on `totalGoldEarned` (0в†’100в†’300в†’700в†’1500в†’3000в†’6000в†’10000)
- `getWeekRevenue()`: 7-day gold from `task.done[dk].drops.gold`
- `renderShop()`: shop level card, gold balance, 2Г—2 material grid, weekly revenue bar chart, 14-day shelf of item cards
- `ITEM_RARITY`: common(easy)/uncommon(normal)/rare(hard)/epic(legendary) with matching border glows
- Shelf capped at 18 items, sorted newest-first

### Step 5 вЂ” Pomodoro = Forge session
Dungeon state extended with `activeWorkTaskId` and `forgeTaskPomoCount`.
- `setForgeTask(taskId)`: links/unlinks a task to the active dungeon
- `tickDungeon`: on each completed 25-min work session в†’ if task linked: `+5g` partial drop, increment `forgeTaskPomoCount`, show notif
- Task cards: `вљ’` forge button appears on undone trabajo tasks when dungeon is running; active task gets amber border glow + live mini-progress bar + session tally
- Dungeon explore view: active task name shown in the pomodoro section
- Task card chips: trabajo cards show `рџЄ™Xg + emojiГ—qty` instead of empty stat chips (both pending and done states)
